# Cortex

WSB ticker mention tracker. Aggregates crowd-sentiment data into a live dashboard.

## Design Context

### Users
Ray (UX designer, builder, occasional trader) and trader / investor friends who get a shared URL. Used as a heads-up display while doing other things (browsing X, looking at charts, in a tab next to a brokerage app). Glanced at frequently, not stared at — so density and at-a-glance comprehension matter more than friendliness or onboarding. Must read as **credible to market-minded people** the first time they see it.

### Brand Personality
- **Three words:** instrumented, opinionated, fast.
- **Voice:** terse, factual, no marketing fluff. Status-line readouts. Tabular data. Reads like a single-purpose monitoring instrument.
- **Emotional goal:** the user should feel like they're watching the crowd's mind through a precision tool — not a consumer app.

### Aesthetic Direction
- **Reference:** Bloomberg Terminal × old amber-phosphor trading station × Teenage Engineering chassis. Dark, dense, monospace, instrumented.
- **Anti-reference:** SaaS template (no left sidebar nav, no top breadcrumbs, no card-grid hero, no "Welcome back, Ray 👋"). Also not Robinhood-style green-on-white, not crypto neon, not glassmorphism.
- **Mode:** dark only. Audience uses this alongside other dark trading interfaces; light mode would clash and read consumer-y.
- **Typography:** Monaspace family (Krypton for headings/wordmark, Neon for body/tables). Distinctive without being IBM-Plex/JetBrains default. All-mono is the move for terminal vibe.
- **Color:** warm-tinted neutrals (slight amber tint in all grays), single warm-amber accent (#c9a86a-ish in OKLCH), muted sage for up, muted dusty rose for down. No bright green, no neon.

### Design Principles
1. **Density over generosity.** Show everything. Terminal users want to see the data without scrolling. Tight padding, packed grids, thin 1px dividers instead of cards-with-shadows.
2. **Instrument cues, not decoration.** Status LEDs, model-number labels, section headers as channel-strip ticks, monospace tabular figures — every flourish should look like it's *displaying state*, not styling for its own sake.
3. **Hierarchy via weight + case, not color.** Color is reserved for signal (up/down/accent). Heading vs body is differentiated by uppercase letterspacing + weight, not size jump or color shift.
4. **Current-state indication everywhere.** Live dot, last-updated timestamp, next-pull countdown, sector buzz fills — always show what's true right now.
5. **No glow, no gradient text, no glassmorphism, no decorative sparklines.** Sparklines stay because they convey real data. Borders on cards are ≤1px. Pure black/white never appears.
